CHƯƠNG 1 - HIGH-LEVEL DESIGN (HLD)
1.1 Architectural Principles
Với vai trò Solution Architect, toàn bộ thiết kế AMS tuân thủ các nguyên tắc sau, được áp dụng nhất quán từ tầng presentation xuống tầng data:
Nguyên tắcÁp dụng trong AMSPattern cụ thểSeparation of ConcernsMỗi layer chỉ biết layer kề nó. UI không truy cập DB trực tiếp.Feature Folders: /Features/CT/Commands, /Features/CT/QueriesSingle ResponsibilityMỗi class/service chỉ làm 1 việc. VoucherService không chứa tax logic.TaxEngine tách riêng khỏi VoucherServiceDependency InversionBusiness layer không phụ thuộc infrastructure. IVoucherRepository interface.IVoucherRepository → SqlServerVoucherRepo / PgVoucherRepoDefense in DepthBảo mật nhiều lớp: Windows Auth + RBAC + HTTPS + AuditLog + DB DENY.5 lớp bảo mật độc lập, 1 lớp fail không ảnh hưởng lớp khácFail FastValidate đầu vào sớm nhất có thể. Không xử lý dữ liệu sai.FluentValidation tại API boundary; DB constraint là lớp cuốiIdempotencyHangfire jobs có thể chạy lại không gây duplicate. SP idempotent.SP_AutoSwitchTaxRates kiểm tra trước khi UPDATEObservabilityMọi exception, slow query, state transition đều có log có thể query.Serilog structured → SQL sink + File. CorrelationId trên mọi requestZero-Trust DBApp không tin tưởng DB sẽ từ chối bad data. Validate ở 3 tầng.FluentValidation → Domain validation → DB constraint

2.3 Tax Engine - Design Detail
TaxEngine là component phức tạp nhất, được thiết kế hoàn toàn stateless (pure functions) và tham số hoá 100% từ DB:

Tự switch theo EffectiveFrom/To.(amount, taxRateKey, date) → {taxBase, vatAmount, rate}TNDNCalculatorICITCalculator1. Tổng DT năm → xác định TS (15% hoặc 20%). 2. Lọc CP không được trừ (CITAdjFlag). 3. TNCT = DT - CP_được_trừ. 4. Thuế = TNCT × TS.(revenue, expenses[], fiscalYear) → {taxableIncome, rate, tax}TNCNCalculatorIPITCalculator1. Load PITBrackets theo date. 2. GTGC = self + n×dependent. 3. TNTT = salary - BHXH - GTGC. 4. Biểu lũy tiến 5 bậc với QuickDeduction.(salary, bhxh, selfAllowance, dependents, date) → {taxableIncome, tax, breakdown[]}TTDBCalculatorIExciseTaxCalculator1. 2.4 Voucher State Machine - Implementation

📌 Sequence diagram trên mô tả luồng tính khấu trừ TNCN tháng - quy trình phức tạp nhất liên quan Tax Engine, 2 bảng DB tham số, và EF Core batch insert.

State Machine Voucher được implement qua Status enum + Guard Methods trong Voucher entity:
TransitionMethodGuard ConditionSide EffectsDRAFT → PENDINGvoucher.Submit()Lines.Count ≥ 2. ΣDebit = ΣCredit. FiscalPeriod.Status = OPEN.Raise VoucherSubmittedEvent → SignalR notify KeToanTruongPENDING → APPROVEDvoucher.Approve(approverId)CurrentUser has KeToanTruong role.AuditLog entry. Raise VoucherApprovedEvent.PENDING → DRAFTvoucher.Reject(reason)CurrentUser has KeToanTruong role. Reason not empty.AuditLog. Raise VoucherRejectedEvent → notify KeToanVien.APPROVED → POSTEDvoucher.Post()FiscalPeriod.Status = OPEN. All referenced Accounts IsActive=true.Call LedgerService.Post(). Insert LedgerEntries (batch). Update account balances. AuditLog.POSTED → REVERSEDvoucher.Reverse(reversedById)FiscalPeriod.Status = OPEN.Create new Voucher (reversed lines, DRAFT). Link RelatedVoucherId. AuditLog.

2.5.2 Key Design Decisions
Quyết địnhLý doTrade-offGUID (NEWSEQUENTIALID / gen_random_uuid) cho PK entity chínhCho phép client-side ID generation. Tránh INT overflow khi multi-year data. ID không đoán được (security).Index fragmentation trên SQL Server → dùng NEWSEQUENTIALID() thay NEWID() để giảm fragmentation. PG: gen_random_uuid() là OK.BIGINT IDENTITY cho AuditLogs PKAuditLogs cần thứ tự thời gian tuyệt đối. GUID không đảm bảo sort order. BIGINT đủ cho hàng tỷ rows.AuditLogs không distribute được nếu sau này cần sharding - acceptable vì on-premise.Soft Delete (IsDeleted) cho mọi entity chínhLuật KT yêu cầu bảo quản 10 năm. Hard delete vi phạm. Audit trail không đầy đủ nếu row bị xoá.Query phải thêm WHERE IsDeleted=0 (EF Core Global Query Filter giải quyết tự động).EF Core Global Query Filter cho Soft Delete và FiscalPeriodDeveloper không cần nhớ WHERE IsDeleted=0. DRY principle. Giảm bug.Phải override filter khi cần xem deleted records (admin). HasQueryFilter().IgnoreQueryFilters()JSONB (PG) / NVARCHAR(MAX) (SS) cho ScheduledRatesExciseTaxItems cần lưu lộ trình 5 năm. Relational table sẽ cần join phức tạp. JSON flexible hơn.PG: JSONB có index → query nhanh. SS: JSON không có native index → chỉ query full scan. Acceptable vì table nhỏ.Outbox Pattern cho Domain EventsDomain Events (VoucherPosted) cần đảm bảo reliable delivery tới Hangfire. Nếu dùng memory event bus, crash sau SaveChanges → event lost.Thêm bảng OutboxMessages trong acc schema. Hangfire poller đọc và publish. Eventual consistency nhưng reliable.Optimistic Concurrency (RowVersion/xmin)Prevent lost updates khi 2 user cùng edit Voucher. SS: RowVersion automatic. PG: dùng xmin system column.EF Core handle concurrency exception → return 409 Conflict với message friendly.

2.7 Security Design - Defense in Depth
LayerMechanismImplementationFail behaviourL1 - NetworkHTTPS/TLS 1.2+IIS with AD Certificate Services cert. TLS 1.0/1.1 disabled. HSTS header. HTTP → HTTPS redirect.Plain HTTP → 301 redirect. Downgrade attack → connection refused.L2 - AuthenticationWindows Kerberos/NTLMIIS Windows Authentication enabled. Anonymous disabled. Browser negotiates automatically.No valid Windows token → 401 Unauthorized. Log to AuditLog.L3 - AuthorizationRBAC + ASP.NET Core Policy[Authorize(Roles=...)] on all controllers/actions. Policy-based for fine-grained (e.g. only KeToanTruong can Post).Invalid role → 403 Forbidden. AuditLog entry with attempted action.L4 - Input ValidationFluentValidationAll Commands/Queries validated. Reject invalid data early. SQL injection impossible via EF Core parameterized queries.400 Bad Request with structured error details. No data reaches DB.L5 - DatabaseDeny + AppRoleDENY UPDATE/DELETE on audit.AuditLogs. Minimal DB permissions for app account. SQL Server: Windows SSPI only. PG: scram-sha-256.DB error returned as 500 (internal). Never expose DB error messages to client.L6 - AuditImmutable AuditLogEF Core Interceptor: every SaveChanges → append AuditLog. BIGINT IDENTITY ensures ordering. No delete allowed.AuditLog write failure → rollback entire transaction. Better to fail than lose audit trail.L7 - SessionCSRF + TimeoutASP.NET Core DataProtection Anti-forgery token on all POST. 30-min session timeout.
