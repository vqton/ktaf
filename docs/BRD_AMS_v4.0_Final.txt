3.3 So Sánh SQL Server 2022 vs PostgreSQL 16
🆕 MỚI: Cả hai lựa chọn đều expose IDENTICAL schemas (acc, tax, audit, cfg, rpt). Application code không cần thay đổi - chỉ EF Core provider và connection string khác nhau.

Tiêu chíSQL Server 2022 (Option A)PostgreSQL 16 (Option B)Phiên bảnDeveloper Edition (dev) → Standard (prod)PostgreSQL 16.x (Community - miễn phí)LicenseTrả phí (Standard ~$900/core hoặc $3,700/server)Miễn phí - PostgreSQL LicenseAuthenticationWindows Auth (SSPI) - BẮT BUỘC. Không có SQL Login.Windows Auth (SSPI/pg_ident) HOẶC Username+Password (pg_hba.conf md5/scram-sha-256)Connection StringServer=.;DB=AMS_DB;Integrated Security=SSPI;Host=localhost;DB=ams_db;Username=ams_user; (pg login) hoặc Integrated Security=true; (SSPI)EF Core ProviderMicrosoft.EntityFrameworkCore.SqlServerNpgsql.EntityFrameworkCore.PostgreSQLStored ProceduresT-SQL (SP_PostVoucher, SP_CloseMonth...)PL/pgSQL (tương đương T-SQL, cú pháp khác)Backup & RestoreSQL Server Agent + Maintenance PlanspgAgent hoặc pg_cron + pg_dump/pg_restoreCollation tiếng ViệtVietnamese_CI_AS trên từng columnCollation: vi-x-icu (ICU collation) hoặc C.UTF-8Full-Text SearchSQL Server FTS built-inpg_trgm extension (LIKE fast), tsvector/tsqueryJSON SupportNVARCHAR(MAX) + JSON functionsJSONB native (indexable, nhanh hơn)MonitoringSQL Server Profiler, Activity Monitorpg_stat_activity, pgBadger, pgAdmin 4Windows IntegrationNative - SQL Server đi kèm WindowsCài riêng. pg_ident.conf map AD user → pg userKhuyến nghịƯu tiên nếu team đã quen SQL ServerƯu tiên nếu muốn zero license cost, hoặc sau này cần scale Linux

3.4 Cấu Hình Database - Hướng Dẫn Admin
Option A: SQL Server 2022 - Cấu Hình
Kết nối bắt buộc dùng Windows Authentication. Không được cấu hình SQL Login cho bất kỳ user nào.
BướcThao tácChi tiết1appsettings.json"DatabaseProvider": "SqlServer" "ConnectionStrings:DefaultConnection": "Server=.;Database=AMS_DB;Integrated Security=SSPI;TrustServerCertificate=True"2SQL Server AuthAuthentication Mode: Windows Authentication ONLY. Không enable Mixed Mode.3AD Group → SQL LoginTạo SQL Server Login từ AD Group: CREATE LOGIN [DOMAIN\AMS_App] FROM WINDOWS4DB PermissionGRANT db_datareader, db_datawriter, EXECUTE TO [DOMAIN\AMS_App]5Hangfire Job StoreHangfire tự tạo schema trong AMS_DB. SQL Server Agent cho backup riêng biệt.6Developer Ed noteDeveloper Edition KHÔNG dùng production. Nâng cấp Standard Edition trước go-live.
Option B: PostgreSQL 16 - Cấu Hình
PostgreSQL hỗ trợ cả hai phương thức xác thực - admin chọn theo chính sách bảo mật của doanh nghiệp.
Auth MethodMô tảCấu hình pg_hba.confappsettings.jsonWindows Auth (SSPI)Dùng Kerberos/NTLM qua AD - không cần nhập mật khẩu PGhost  all  all  127.0.0.1/32  sspi  include_realm=0  krb_realm=DOMAIN.LOCAL"DatabaseProvider": "PostgreSql" Integrated Security=true;Host=localhost;Database=ams_dbpg Login (Username+Password)Tạo user riêng trong PostgreSQL - phù hợp khi AD chưa cấu hình SSPIhost  all  ams_user  127.0.0.1/32  scram-sha-256"DatabaseProvider": "PostgreSql" Host=localhost;Database=ams_db;Username=ams_user;Password=<secret>Cả 2 (Hybrid)AD user dùng SSPI; service account dùng pg loginCấu hình cả 2 dòng pg_hba theo thứ tự ưu tiênChọn theo user type - EF DbContext factory
BướcThao tác PostgreSQLChi tiết1appsettings.json"DatabaseProvider": "PostgreSql" (switch EF provider). Connection string tương ứng auth method.2Tạo databaseCREATE DATABASE ams_db WITH ENCODING='UTF8' LC_COLLATE='vi_VN.UTF-8' TEMPLATE=template0;3Tạo user (pg login)CREATE ROLE ams_user LOGIN PASSWORD 'strong_password'; GRANT CONNECT ON DATABASE ams_db TO ams_user;4Tạo schemasCREATE SCHEMA acc; CREATE SCHEMA tax; CREATE SCHEMA audit; CREATE SCHEMA cfg; CREATE SCHEMA rpt;5EF Migrationdotnet ef database update -- sẽ tạo toàn bộ tables theo migration scripts6Collation tiếng ViệtĐặt collation cho cột Name, Description: ALTER TABLE... ALTER COLUMN name TYPE varchar COLLATE "vi-x-icu"7pgAgent backupCấu hình pgAgent job: pg_dump -Fc ams_db > /backup/ams_$(date +%Y%m%d).dump hàng đêm8SSPI config (nếu dùng)Sửa pg_ident.conf: map AD username → pg username. Khởi động lại PostgreSQL service.
